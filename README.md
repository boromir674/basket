# Euroleague Sankey Spike — develop snapshot

Clean starting point for continuing the spike on a corporate Windows laptop with Docker only.

## Files

- `build_from_euroleague_api.py`
- `poss-flow-map-multi-drilldown-real-data.html`
- `Dockerfile`
- `docker-compose.yml`
- `sample_multi_drilldown_real_data.json`

## Build

```bash
docker build -t euroleague-sankey .
```

## Quick demo (primary entry point)

This is the minimal one-liner you can use day-to-day. It:

- Runs a small curated batch of games.
- Writes one JSON per game.
- Updates `multi_drilldown_real_data.json` so the HTML viewer picks up the
	latest game automatically.
- Runs validation over all generated files.

```sh
docker run --rm -v "${PWD}:/app" -w /app euroleague-sankey demo
```

With Docker Compose:

```sh
docker compose run --rm demo
```

## Run pipeline + validation (single game)

PowerShell:

```sh
docker run --rm -v "${PWD}:/app" -w /app euroleague-sankey \
	run_pipeline_and_validate --seasoncode E2021 --gamecode 54 \
	--output-dir /app --output-pattern multi_drilldown_real_data_{seasoncode}_{gamecode}.json
```

cmd.exe:

```cmd
docker run --rm -v "%cd%:/app" -w /app euroleague-sankey ^
	run_pipeline_and_validate --seasoncode E2021 --gamecode 54 ^
	--output-dir /app --output-pattern multi_drilldown_real_data_{seasoncode}_{gamecode}.json
```

## Run pipeline + validation (multiple games)

PowerShell:

```sh
docker run --rm -v "${PWD}:/app" -w /app euroleague-sankey \
	run_pipeline_and_validate --seasoncode E2021 \
	--gamecode 54,55,56 \
	--output-dir /app --output-pattern multi_drilldown_real_data_{seasoncode}_{gamecode}.json
```

## Serve HTML


```sh
docker run --rm -p 8080:8080 -v "${PWD}:/app" -w /app python:3.12-slim python -m http.server 8080
```

Open:

```text
http://localhost:8080/poss-flow-map-multi-drilldown-real-data.html
```

Or use the landing page to switch games:

```text
http://localhost:8080/index.html
```

The viewer always reads JSON from disk, so any time you rerun the
pipeline (e.g. `docker compose run demo` or a custom `pipeline` command)
and overwrite the JSON files, a browser refresh will pick up the
updated data.

## Rebuild the game manifest from existing files

If you already have a bunch of `multi_drilldown_real_data_*.json` files
locally and just want the game switcher UI to see them without running
`season_sync` over a whole season, you can rebuild `games_manifest.json`
from disk only:

```sh
docker run --rm -v "${PWD}:/app" -w /app euroleague-sankey \
	rebuild_manifest --seasoncode E2021 --output-dir /app
```

This is additive and backward-compatible: it only rewrites
`games_manifest.json` based on the JSON files already present.

## Development: Critical Data Flows

### Raw Data Handling
- **Source**: Raw data is fetched using `build_from_euroleague_api.py`.
- **Storage**: Saved in the `assets/` directory.
- **Purpose**: Acts as the intermediate cache for further processing.

### Processed Data Handling
- **Source**: Processed data is generated by `pipeline_runner.py`.
- **Storage**: Saved in the `assets/processed/` directory.
- **Purpose**: Provides the data consumed by the UI.

### UI Integration
- **Source**: The UI dynamically reads processed JSON files from `assets/processed/`.
- **Default File**: If no file is specified, the UI defaults to `assets/processed/multi_drilldown_real_data.json`.
- **Customization**: Users can switch between multiple games by passing `?file=<filename>` in the URL.

### Validation
- **Pipeline Validation**: Ensures correctness of processed files.
- **Manifest Rebuild**: `games_manifest.json` is rebuilt to reflect available processed files, enabling the game switcher UI.
